# Техническое задание
## Habit Tracker - Web приложение для трэкинга привычек

---

## 1. Описание проекта

**Название:** Habit Tracker
**Тип:** Web-приложение (PWA)
**Платформы:** Мобильные браузеры (iOS Safari, Android Chrome), десктопные браузеры
**Основная фокус:** iPhone и iPhone Pro Max

Приложение для отслеживания ежедневных привычек (ритуалов) с удобным календарным интерфейсом, ведением статистики и облачной синхронизацией через Firebase. UI с поддежкой выбора языка. На первых этапах нужен Ru(по умолчанию), En, De. Реализуется через словарь.  

---

## 2. Функциональные требования

### 2.1 Основные возможности

#### A. Главная страница (Dashboard)
- **Календарное представление:**
  - Режим просмотра: текущая неделя (по умолчанию)
  - Опциональный режим: месячный календарь
  - Переключение между неделями/месяцами (кнопки "Назад" и "Далее")
  - Отображение дня недели и даты

- **Ежедневные привычки:**
  - Для каждого дня отображается список привычек с эмодзи
  - Визуальная индикация статуса: выполнена/не выполнена
  - Интерактивность: клик по привычке = переключение статуса
  - Двойной клик или долгий клик = сброс случайно поставленной отметки

- **Конфигурируемое редактирование прошлых дней:**
  - По умолчанию: редактировать можно только сегодня и один день назад
  - Опция в настройках: увеличить количество дней для редактирования
  - Защита от изменения старых дней (для честности отслеживания)

#### B. Страница добавления/редактирования привычки
- **Форма добавления привычки:**
  - Самые основные привычки должны быть предсозданы и отображаться именно на том языке, на котором у пользователя в данный момент интерфейс
  - Поле ввода названия привычки
  - Выбор эмодзи из предустановленного списка (55+ вариантов)
  - Цветовая категория (опционально): Здоровье, Продуктивность, Образование, Финансы, Саморазвитие и т.д.
  - Описание привычки (опционально)
  - Время напоминания (опционально)
  - Кнопки: "Сохранить" и "Отмена"

- **Редактирование привычки:**
  - Возможность изменить название, эмодзи, категорию
  - Опция удаления привычки с подтверждением
  - Историцеские данные об удалённой привычке не должны удаляться
  - В настройках нужно отображать привычки которые пользователь удалил, с возможностью вернуть привычку и отображать кооректно верь трэк исторических данных по привычке
  - История изменений (опционально)

#### C. Страница статистики (Analytics)
- **Общие метрики:**
  - Процент выполнения всех привычек (общая статистика)
  - Полосы прогресса для каждой привычки (за текущую неделю/месяц)
  - Лучшая привычка (наибольший процент выполнения)
  - Самая "ленивая" привычка (наименьший процент)

- **Детальная статистика:**
  - График выполнения по дням (последние 30 дней)
  - Строки достижений (серии выполнения подряд)
  - Калорийность усилий (количество выполненных привычек за день)

- **Временные фильтры:**
  - Неделя
  - Месяц
  - Пользовательский диапазон дат

#### D. Страница профиля и настроек
- **Профиль пользователя:**
  - Аватар (опционально)
  - Email
  - Дата создания аккаунта
  - Общая статистика (всего дней отслеживания, всего привычек)

- **Настройки:**
  - Количество дней для редактирования прошлых привычек (1-30 дней)
  - Режим календаря (неделя/месяц)
  - Напоминания (включить/отключить)
  - Тема оформления (светлая/темная) - опционально
  - Удалённые привычки, с возможностью вернуть привычку
  - Язык интерфейса
  - Выход из аккаунта

## 4. Архитектура данных (Firestore Schema)

### 4.1 Коллекции и документы

#### Коллекция: `users`
```
users/
├── {userId}/
│   ├── email: string
│   ├── displayName: string (опционально)
│   ├── avatar: string (URL, опционально)
│   ├── createdAt: timestamp
│   ├── settings: object
│   │   ├── editableDaysCount: number (default: 1)
│   │   ├── calendarView: string (week/month)
│   │   ├── notificationsEnabled: boolean
│   │   ├── theme: string (light/dark)
│   │   └── language: string (ru/en)
│   └── stats: object
│       ├── totalDaysTracked: number
│       ├── totalHabits: number
│       └── createdHabitCount: number
```

#### Коллекция: `habits` (принадлежит user)
```
users/{userId}/habits/
├── {habitId}/
│   ├── name: string (название привычки)
│   ├── emoji: string (одиночный эмодзи)
│   ├── category: string (Health, Productivity, Education, Finance, SelfDevelopment)
│   ├── description: string (опционально)
│   ├── createdAt: timestamp
│   ├── updatedAt: timestamp
│   ├── isActive: boolean (default: true)
│   ├── reminderTime: string (опционально, HH:mm формат)
│   ├── color: string (опционально, hex code для категории)
│   └── statistics: object
│       ├── totalCompleted: number
│       └── currentStreak: number
```

#### Коллекция: `dailyHabits` (запись о выполнении)
```
users/{userId}/dailyHabits/
├── {YYYY-MM-DD}/
│   ├── date: string (YYYY-MM-DD)
│   ├── dayOfWeek: number (0-6, 0 = понедельник)
│   ├── habits: object
│   │   ├── {habitId}: boolean (true = выполнена, false = не выполнена)
│   │   ├── {habitId}: boolean
│   │   └── ...
│   └── updatedAt: timestamp
```

#### Коллекция: `habitHistory` (опционально, для истории изменений)
```
users/{userId}/habitHistory/
├── {historyId}/
│   ├── habitId: string
│   ├── action: string (created/updated/deleted)
│   ├── changedFields: object (какие поля изменились)
│   ├── timestamp: timestamp
│   └── oldValues: object
```

### 4.2 Индексы Firestore (для оптимизации запросов)
- `users/{userId}/dailyHabits`: индекс по `date` (для быстрого поиска по дате)
- `users/{userId}/habits`: индекс по `createdAt` (для сортировки)

## 6. Функциональная логика

### 6.1 Основной сценарий использования (User Flow)

```
1. Первый запуск
   ├─ Загрузка приложения
   ├─ Проверка аутентификации
   ├─ Если не авторизован → страница входа/регистрации (FirebaseUI)
   └─ Если авторизован → Dashboard (главная страница)

2. Dashboard (Главная)
   ├─ Загрузить текущую неделю (или месяц по настройкам)
   ├─ Загрузить привычки пользователя
   ├─ Загрузить статусы выполнения для каждого дня
   ├─ Отрендерить календарь и привычки
   └─ Слушать клики на привычках:
      ├─ Клик → переключить статус (выполнена/не выполнена)
      └─ Обновить в Firestore

3. Добавление привычки
   ├─ Перейти на страницу "Add Habit"
   ├─ Заполнить форму (название, эмодзи, категория, описание)
   ├─ Сохранить в Firestore (коллекция habits)
   └─ Вернуться на Dashboard

4. Просмотр статистики
   ├─ Перейти на страницу "Statistics"
   ├─ Загрузить данные за выбранный период
   ├─ Вычислить метрики:
   │  ├─ % выполнения по каждой привычке
   │  ├─ Серии выполнения (streaks)
   │  └─ Общий % выполнения
   └─ Отрендерить графики и диаграммы

5. Редактирование прошлых дней
   ├─ Проверить количество дней назад (по настройкам)
   ├─ Если дата <= editableDaysCount дней назад → разрешить редактирование
   └─ Иначе → показать блокировку
```

### 6.3 Бизнес-логика

#### Проверка редактируемости дня
```javascript
function isDateEditable(dateString, editableDaysCount) {
  const selectedDate = new Date(dateString);
  const today = new Date();
  today.setHours(0, 0, 0, 0);
  
  const daysAgo = Math.floor((today - selectedDate) / (1000 * 60 * 60 * 24));
  
  return daysAgo <= editableDaysCount;
}
```

#### Расчёт процента выполнения
```javascript
function calculateCompletionPercentage(habit, dailyRecords) {
  const completedDays = dailyRecords.filter(record => 
    record.habits[habit.id] === true
  ).length;
  
  const totalDays = dailyRecords.length;
  return (completedDays / totalDays) * 100;
}
```

#### Расчёт серии выполнения (streak)
```javascript
function calculateStreak(habit, dailyRecords) {
  let streak = 0;
  const today = new Date();
  
  for (let i = 0; i < 365; i++) {
    const date = new Date(today);
    date.setDate(date.getDate() - i);
    const dateString = date.toISOString().split('T')[0];
    
    const record = dailyRecords.find(r => r.date === dateString);
    if (record && record.habits[habit.id]) {
      streak++;
    } else {
      break;
    }
  }
  
  return streak;
}
```

## 8. Технические требования и ограничения

### 8.1 Производительность
- **Загрузка страницы:** < 2 сек (на 3G)
- **Отклик на действие:** < 200ms
- **FCP (First Contentful Paint):** < 1.5 сек
- **Оптимизация изображений:** WebP, оптимизированные размеры
- **Минификация:** CSS и JS должны быть минифицированы

### 8.2 Безопасность
- **HTTPS:** Обязателен
- **Firebase Security Rules:** 
  - Пользователь может читать/писать только свои данные
  - Валидация данных на уровне Firestore
- **CSP (Content Security Policy):** Устанавливать для защиты от XSS
- **Запрет на сохранение паролей:** FirebaseUI управляет безопасностью

### 8.3 Браузерная поддержка
- iOS Safari: 12+
- Chrome: 80+
- Firefox: 75+
- Edge: 80+

### 8.4 Оффлайн функциональность (опционально)
- Сохранять статусы в localStorage
- Синхронизировать с облаком при восстановлении соединения
- Service Worker для offline-first подхода

## 13. Глоссарий

| Термин | Описание |
|--------|----------|

| **Статус привычки** | Выполнена или не выполнена в конкретный день |
| **Streak** | Количество дней подряд, когда привычка была выполнена |

## Заключение
